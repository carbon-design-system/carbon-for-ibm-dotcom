## Getting started

### Quick start
##### (Note: Fade effect won't be noticeable unless the page is long enough to scroll through)

In order to use the `<dds-fade-in-out>` component, it needs to be wrapped around the elements that need to be targeted,
passing in a querystring containing the exact identifiers to control which elements it should target.

Here's a simple example to get started:

#### JS

```javascript
import '@carbon/ibmdotcom-web-components/es/components/fade-in-out/fade-in-out.js'
import '@carbon/ibmdotcom-styles/scss/components/scroll-into-view/_scroll-into-view.scss';
```

#### HTML

```html
<dds-fade-in-out selector-targets='.targeted-class'>
  <div class='targeted-class'>
    <p>This content will fade in upon scrolled into view.</p>
  </div>

  <div>
    <p>This content is not included in the selector, it will not fade.</p>
  </div>
</dds-fade-in-out>
```

### CSS

An optional custom delay can be set within targeted class in the application's CSS code as such:

```css
.targeted-class {
  --dds--fade-in-out-delay: 250ms;
}
```
