## Getting started

Here's a quick example to get you started.

### JS

```javascript
import ArrowRight20 from '@carbon/icons-react/es/arrow--right/20.js';
import C4DCard from '@carbon/ibmdotcom-web-components/es/components-react/card/index';
import C4DCarousel from '@carbon/ibmdotcom-web-components/es/components-react/carousel/index';

function App() {
  return (
    <C4DCarousel>
      <C4DCard href="https://www.ibm.com/standards/carbon">
        <C4DCardHeading>Lorem ipsum dolor sit amet</C4DCardHeading>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. Aenean et ultricies
        est.
        <C4DCardFooter>
          <ArrowRight20 slot="icon" />
        </C4DCardFooter>
      </C4DCard>
      <C4DCard href="https://www.ibm.com/standards/carbon">
        <C4DCardHeading>Lorem ipsum dolor sit amet</C4DCardHeading>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. Aenean et ultricies
        est. Mauris iaculis eget dolor nec hendrerit. Phasellus at elit sollicitudin,
        sodales nulla quis, consequat libero.
        <C4DCardFooter>
          <ArrowRight20 slot="icon" />
        </C4DCardFooter>
      </C4DCard>
    </C4DCarousel>
  );
}
```

### SCSS

While `<c4d-carousel>` has `page-size` attribute, `<c4d-carousel>` also looks at
`--cds--carousel--page-size` CSS custom property to determine the number of
cards per page. This is useful for setting different number of cards for
different breakpoints.

```scss
@use '@carbon/styles/scss/breakpoint';

cds-carousel {
  --cds--carousel--page-size: 1;

  @include breakpoint(md) {
    --cds--carousel--page-size: 2;
  }

  @include breakpoint(lg) {
    --cds--carousel--page-size: 3;
  }
}
```

Another CSS custom property `<c4d-carousel>` looks at is
`--cds--carousel--overflow-hint-size`. If `0rem` is set there, the first card at
the next page as a hint of page overflow will be hidden.
