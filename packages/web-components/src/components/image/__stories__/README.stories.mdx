# Image

> The image component is used to embed images within content and can display a
> specific responsive image depending on viewport size.
> Image items are used to change how the image behaves at different screen sizes.
> Below you'll find an example with and without media query behaviors.

## Getting started

Here's a quick example to get you started.

### JS (via import)

```javascript
import '@carbon/ibmdotcom-web-components/es/components/image/index.js';
```

<Description markdown={`${cdnJs({ components: ['image'] })}`} />

<Description markdown={`${cdnCss()}`} />

### HTML

```html
// Example without media queries
<dds-image alt="Image alt text">
  <dds-image-item href="https://fpoimg.com/672x336?text=2:1&bg_color=ee5396&text_color=161616"> </dds-image-item>
</dds-image>

// Example with media queries
<dds-image alt="Image alt text">
  <dds-image-item media="(min-wiidth:672px)" href="https://fpoimg.com/672x336?text=2:1&bg_color=ee5396&text_color=161616">
  </dds-image-item>
  <dds-image-item media="(min-wiidth:400px)" href="https://fpoimg.com/672x336?text=2:1&bg_color=ee5396&text_color=161616">
  </dds-image-item>
  <dds-image-item media="(min-wiidth:320px)" href="https://fpoimg.com/672x336?text=2:1&bg_color=ee5396&text_color=161616">
  </dds-image-item>
</dds-image>
```

## Stable selectors

See [our README](https://github.com/carbon-design-system/carbon-for-ibm-dotcom/tree/master/packages/web-components#stable-selectors-for-analytics-and-integratione2e-testing-in-web-components) to see how Web Components selector and `data-autoid` should be used.

| Web Components selector | Compatibility selector          | Description |
| ----------------------- | ------------------------------- | ----------- |
| `<dds-image>`           | `data-autoid="dds--image"`      | Component   |
| `<dds-image-item>`      | `data-autoid="dds--image-item"` | Interactive |
