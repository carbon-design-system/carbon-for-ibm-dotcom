## Getting started

Here's a quick example to get you started.

### JS (via import)

```javascript
import '@carbon/ibmdotcom-web-components/es/components/leaving-ibm/index';
```

<Description markdown={`${cdnJs({ components: ['leaving-ibm'] })}`} />

<Description markdown={`${cdnCss()}`} />

### HTML

```html
<!-- Add component to document -->
<c4d-leaving-ibm-container></c4d-leaving-ibm-container>

...

<!-- Trigger modal on click -->
<c4d-link-with-icon
  href="https://www.carbondesignsystem.com/all-about-carbon/what-is-carbon/"
  data-leaving-ibm>
  Learn more about the Carbon Design System
</c4d-link-with-icon>
```

> ðŸ’¡ Give your link the `data-leaving-ibm` attribute to trigger the modal on
> click. Alternatively, you can use the
> [custom events available to the modal](https://web-components.carbondesignsystem.com/?path=/docs/components-modal--default#cds-modal-attributes-and-properties)
> to create custom triggering behavior.

## Stable selectors

See
[our README](https://github.com/carbon-design-system/carbon-for-ibm-dotcom/tree/main/packages/web-components#stable-selectors-for-analytics-and-integratione2e-testing-in-web-components)
to see how Web Components selector and `data-autoid` should be used.

| Web Components selector   | Compatibility selector                 | Description |
| ------------------------- | -------------------------------------- | ----------- |
| `<c4d-leaving-ibm-modal>` | `data-autoid="c4d--leaving-ibm-modal"` | Component   |
| `cds--leaving-ibm-cta`    | `data-autoid="c4d--leaving-ibm-cta"`   | Interactive |
